# GuessAndWin

A small browser-based simulator for comparing Poisson vs empirical goal distributions and optimizing expected-value (EV) for match score predictions.

Features
- Toggle between Poisson model and an empirical "overs"-based model.
- Sliders with editable numeric inputs for all market percentages (type a number or drag the slider).
- Live EV ranking of scorelines and a highlighted optimal pick.
- Copyable summary of current inputs (one-click copy) for easy sharing or logging.

Quick start
1. Open the project folder in a terminal.
2. Start a simple static server (recommended):

```bash
python -m http.server 8000
# then open http://localhost:8000/index.html in your browser
```

Alternatively open `index.html` directly in the browser, but some browsers restrict local module loading — using a local server avoids that.

Files
- `index.html` — main HTML and styles.

Customization notes
- The empirical model currently groups goals into buckets (0,1,2,3+). To support higher granularity (e.g. 4+ or 5+), edit `GuessAndWinApp.js`:
  - `currentMaxGoals` controls how many explicit goal values are shown (Poisson uses 5 by default, empirical uses 3 meaning `3+`).
  - Add additional `pOver3.5`, `pOver4.5`, ... sliders and update the empirical distribution construction if you want explicit buckets for 3,4,5.
- The `SliderWithNumber` component in `GuessAndWinApp.js` centralizes the range + numeric input behavior.
